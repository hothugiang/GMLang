1. Giới thiệu

Ngôn ngữ UPL+ (Universal Programming Language Plus) được xây dựng dựa trên ngôn ngữ UPL trong đề bài, với mục tiêu mở rộng khả năng biểu đạt và hỗ trợ tốt hơn cho việc xây dựng trình biên dịch.

UPL+ vẫn giữ các đặc điểm cốt lõi của UPL như cú pháp đơn giản, số lượng kiểu dữ liệu hạn chế và các cấu trúc điều khiển cơ bản, nhưng được cải tiến thêm một số tính năng như suy luận kiểu (type inference), mở rộng biểu thức logic và thiết kế biểu diễn trung gian nhằm phục vụ quá trình sinh mã.

2. Đặc tả từ vựng (Lexical Specification)

2.1. Từ khóa (Keywords)

begin, end, int, bool, auto, if, then, else, do, while, for, print

2.2. Định danh (Identifier)

IDENTIFIER = [a-zA-Z]+[0-9]*

Quy tắc:

  • Bắt đầu bằng ký tự chữ cái
  • Có thể chứa các chữ cái tiếp theo
  • Nếu có chữ số, chỉ được xuất hiện ở cuối

Ví dụ hợp lệ:

x, abc, var123

Ví dụ không hợp lệ:

1abc, ab1c

2.3. Hằng số (Literals)

  • Số nguyên: 0, 1, 100
  • Giá trị logic: true, false

2.4. Toán tử (Operators)

+, *, >, >=, ==, &&, ||, !, =

2.5. Ký hiệu phân tách (Separators)

; ( ) { }

2.6. Chú thích (Comments)

// chú thích một dòng

/* chú thích nhiều dòng */

3. Đặc tả cú pháp (Syntax Specification)

Cấu trúc chương trình

begin
{
  <khai báo và câu lệnh>
}
end

3.1. Khai báo biến

int x;
bool a;

int y = 5;

auto z = 10;
auto b = true;

3.2. Câu lệnh gán

x = 10;
y = x + 2;

3.3. Câu lệnh điều kiện

if (x > 5) then {
  print(x);
}
if (x > 5) then {
  print(x);
} else {
  print(0);
}

3.4. Vòng lặp

Do – while

do {
  x = x + 1;
} while (x > 5);

For

for (x = 0; x < 10; x = x + 1) {
  print(x);
}

3.5. Lệnh in

print(x);
print(x + 1);

3.6. Biểu thức

(x + 2) * 3 > 10 && !a

3.7. Độ ưu tiên toán tử

Mức Toán tử
1 !
2 *
3 +
4 >, >=, ==
5 &&
6

4. Văn phạm (Grammar – BNF)

Program ::= "begin" Block "end"

Block ::= "{" DeclList StmtList "}"

DeclList ::= Decl DeclList | ε
StmtList ::= Stmt StmtList | ε

Decl ::= Type Identifier ("=" Expr)? ";"

Type ::= "int" | "bool" | "auto"

Stmt ::= AssignStmt
       | IfStmt
       | WhileStmt
       | ForStmt
       | PrintStmt
       | Block

AssignStmt ::= Identifier "=" Expr ";"
AssignExpr ::= Identifier "=" Expr

IfStmt ::= "if" "(" Expr ")" "then" Stmt
         | "if" "(" Expr ")" "then" Stmt "else" Stmt

WhileStmt ::= "do" Stmt "while" "(" Expr ")" ";"

ForStmt ::= "for" "(" AssignExpr ";" Expr ";" AssignExpr ")" Stmt

PrintStmt ::= "print" "(" Expr ")" ";"

Expr ::= Expr "||" AndExpr | AndExpr
AndExpr ::= AndExpr "&&" EqualityExpr | EqualityExpr
EqualityExpr ::= EqualityExpr "==" RelExpr | RelExpr
RelExpr ::= RelExpr (">" | ">=") AddExpr | AddExpr
AddExpr ::= AddExpr "+" MulExpr | MulExpr
MulExpr ::= MulExpr "*" UnaryExpr | UnaryExpr
UnaryExpr ::= "!" UnaryExpr | Primary

Literal ::= INT_LITERAL | BOOL_LITERAL
Primary ::= Identifier | Literal | "(" Expr ")"

5. Phân tích cú pháp (Parser & AST)

Sử dụng Recursive Descent Parser.

Ví dụ AST:

y = x + 2

Assign
├── id: y
└── Add
    ├── x
    └── 2

6. Đặc tả ngữ nghĩa (Semantic Specification)

6.1. Bảng ký hiệu (Symbol Table)

Tên biến Kiểu Phạm vi Index
x int global 1
y int block 2

Cách hoạt động

  • Khi khai báo:

    • Thêm vào Symbol Table
    • Kiểm tra trùng tên

  • Khi sử dụng:

    • Tra cứu bảng
    • Kiểm tra kiểu

  • Khi ra khỏi block:

    • Xóa biến khỏi bảng

Ánh xạ JVM

x → slot 1 → istore_1 / iload_1
y → slot 2 → istore_2 / iload_2

6.2. Phạm vi (Scope)

  • Xác định bởi { }
  • Biến có hiệu lực trong phạm vi khai báo và phạm vi con

6.3. Kiểm tra khai báo

  • Biến phải được khai báo trước khi sử dụng
  • Không được trùng tên trong cùng phạm vi

6.4. Kiểm tra kiểu dữ liệu

  • +, * → int
  • >, >= → int
  • == → int hoặc bool
  • &&, ||, ! → bool

6.5. Suy luận kiểu (Type Inference)

auto x = 5;     // int
auto a = true;  // bool

6.6. Lỗi ngữ nghĩa

int x;
x = true;   // sai kiểu
x = 5;      // chưa khai báo

7. Biểu diễn trung gian (Intermediate Representation)

UPL+ sử dụng máy ảo dạng stack:

PUSH
LOAD
STORE
ADD, MUL
GT, GE, EQ
AND, OR, NOT
JMP, JMP_IF_FALSE
PRINT
HALT

8. Sinh mã (Code Generation)

Ví dụ

Source

int x = 1;
int y = x + 2;
print(y);

Bytecode (UPL-VM)

PUSH 1
STORE x
LOAD x
PUSH 2
ADD
STORE y
LOAD y
PRINT
HALT

Mapping sang JVM

UPL-VM JVM

LOAD x iload

9. Xử lý lỗi (Error Handling)

Các loại lỗi

  • Lỗi từ vựng
  • Lỗi cú pháp
  • Lỗi ngữ nghĩa

Chiến lược

  • Không dừng ngay khi gặp lỗi
  • Bỏ qua đến ; hoặc }

10. Quy trình biên dịch

Source Code (UPL+)
↓
Lexical Analysis
↓
Syntax Analysis (AST)
↓
Semantic Analysis
↓
Intermediate Code (UPL-VM)
↓
Bytecode Generation
↓
Java Bytecode (JVM)

Điểm cải tiến

  • Suy luận kiểu (auto)
  • Biểu thức logic mở rộng
  • IR riêng (UPL-VM)
  • Pipeline compiler nhiều pha

11. Ví dụ full pipeline

Source

int x = 1 + 2;

Token

int, x, =, 1, +, 2

AST

Assign
├── id: x
└── Add
    ├── 1
    └── 2

IR

PUSH 1
PUSH 2
ADD
STORE x

JVM

iconst_1
iconst_2
iadd
istore_1

12. Kết luận

UPL+ là phiên bản cải tiến của UPL với cú pháp đơn giản nhưng mạnh hơn, hỗ trợ biểu thức logic, suy luận kiểu và có thiết kế trình biên dịch đầy đủ.

Ngôn ngữ có thể mở rộng thêm:

  • Function
  • Array
  • Optimization (constant folding)
